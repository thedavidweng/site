# canvas-cli 领域术语表

## 核心守则

**canvas-cli 是 Canvas LMS 网页版的替代品**，额外提供自动化便利和 agent 友好。能参照 Canvas API 做法的自行决定，只询问 UX 相关的决策。

## 核心概念

**canvas-cli** — Canvas LMS 的本地 CLI 工具，用于管理课程、作业、模块、文件、讨论、收件箱等。仓库名 `canvas-cli`，安装后的二进制文件是 `canvas`。

## 用户

**学生** — 需要快速拉取课程工作上下文的人：课程、模块、作业、文件、成绩、收件箱。也需要安全操作：提交作业、上传文件、回复讨论和消息。

**教学团队** — 教师、助教、评分员、课程设计者。需要运营工作流：管理课程、批量下载提交物、评分、发布公告、批量编辑截止日期和模块结构。

**Agent** — 自动化代理（CI、脚本、cron 任务）。需要确定性、可解析、可逆的行为：稳定命令名、稳定 JSON 输出、可预测的退出码、stdout 数据/stderr 诊断分离。

## 命令设计决策

**API 覆盖** — canvas-cli 最终应覆盖学生和教学团队在浏览器中使用的主要 Canvas 工作流：检查课程、拉取模块/作业/文件/页面/公告/讨论/日历/成绩/收件箱、提交作业、参与讨论、发送消息、批量下载资源和提交物、评分、评论、更新内容。

**安全门控** — 三级安全机制：
- `--read-only` — 全局开关，阻止所有远程修改（适合脚本/Agent）
- `--dry-run` — 操作级，只预览不执行（优先级高于 --confirm）
- `--confirm` — 操作级，确认高风险操作

**输出格式** — 三级输出：
- 默认 — 简洁关键信息（表格形式）
- `--full` — 完整信息
- `--json` — 自动化/Agent 用

**原始 API 逃生舱** — `canvas api` 命令允许直接调用任何 Canvas API 端点，当预置命令不覆盖某个场景时使用。

**速率限制** — Canvas API 有速率限制，canvas-cli 自动处理请求成本计算和退避。
